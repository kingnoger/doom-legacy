How to compile Doom Legacy C++

Compiling the SDL version

You need to do the following stuff:
  • Get GCC 3.2, the Gnu Compiler Collection. Linux systems most likely have it installed.
    For Windows32, download and install MinGW 2.0 (a GCC port) and MSYS (BASH shell and utilities).
  • Download and install SDL 1.2.5 development libraries, which include the runtime library (dll).
    Windows binary version installation:
    • Decompress the file i386-mingw32msvc.tar.gz to a temporary directory
    • Copy the 'include' and 'lib' directories to your MinGW directory
    • Copy the file 'SDL.dll' (runtime library) to your windows/SYSTEM directory.
  • Download and install SDL_mixer-1.2.3 development libraries, using the same procedure as above.
    Windows users, note that 'SDL_mixer.h' needs to go to the 'MinGW/include/SDL' directory, not just 'MinGW/include'!
  • Download the doomlegacy_c++ branch from the CVS.
  • Win32: Start the MSYS shell. Go to the doomlegacy_c++ directory.
    • Say 'mkdir objs' to create the object file directory
    • Say 'make depend' to create the dependency files
    • Say 'make OPT=1' or just 'make' to build the executable.
      Omitting the 'OPT=1' parameter adds debugging symbols to the executable.
