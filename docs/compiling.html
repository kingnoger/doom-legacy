How to compile Doom Legacy

Compiling the SDL version

In order to compile Doom Legacy SDL you'll need to have several libraries installed on your system.
Make sure you install the "developer" packages of the libraries, which include both the runtime libraries (DLLs) and the stuff required during compilation (header files and import libraries).

LibraryversionDescription
SDL 1.2.5 Simple DirectMedia Layer. A multiplatform multimedia library.
SDL_mixer 1.2.3 A multichannel mixer library based on SDL.
zlib 1.1.4 A free compression library.
OpenTNL 1.4.0 A free game networking library based on the Torque Network Library.
The OpenTNL distribution contains libtomcrypt, which provides strong encryption.

You might also find the following programs useful:

  • GCC 3.2, the Gnu Compiler Collection which, among other things, is a C/C++ compiler.
    Linux systems most likely already have it installed.
    Windows users should install MinGW 2.0, a GCC port, and MSYS, a collection of POSIX utilities for Win32.
  • Doxygen, an automatic source documentation system.

Finally, download the Doom Legacy source. You can either get the source package or, for the latest code snapshot, checkout the doomlegacy_c++ branch from the CVS.

Compilation and linking

  1. Open a shell window. Go to the doomlegacy_c++ directory.
  2. Say 'make depend' to create the dependency files.
  3. Say 'make OPT=1' or just 'make' to build the executable.
    Omitting the 'OPT=1' parameter adds debugging symbols to the executable.
  4. Say 'make docs' to generate the Doxygen documentation.

Miscellaneous hints

SDL 1.2.5 Windows binary version installation:
  • Decompress the file i386-mingw32msvc.tar.gz to a temporary directory
  • Copy the 'include' and 'lib' directories to your MinGW directory
  • Copy the file 'SDL.dll' (runtime library) to your windows/SYSTEM directory.
