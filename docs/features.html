Doom Legacy features

As of version XXX, the Legacy game engine has been rewritten in C++. A host of features were added to the game. Here is a short summary.

* Implemented but doesn't affect the game yet.
** Partly implemented

Graphics

  • ** general graphic presentations, MD3 support

Sound and Music

The preferred multimedia interface is now SDL, the Simple DirectMedia Layer. This makes Legacy easily portable and offers great multimedia capabilities. Music is produced using the SDL_mixer library. Highlights:

  • Supports MUS, MIDI, MP3 and Ogg Vorbis music
  • Original Doom and **WAV sounds
  • Software 3D sound renderer

Gameplay

Gameplay is nearly the same as in original Doom/Heretic. However, if required, it can be enhanced with the following features:

  • Advanced teamplay. Deathmatch, co-op, team deathmatch...
  • Weapon groups á la Half-Life
  • Any creature can be played, not just Marines or Heretics
  • Customizable HUD (using HUD widgets)

Editing

The Doom map format is bad. It is hard to expand and relies in many cases on properties hard-coded in the original Doom executable. Things were improved a bit by Hexen, which extended the Doom map format and added the MAPINFO lump, which can be used to create a custom map ordering within the WAD.

Legacy improved the map format further by introducing the MapInfo (or LevelInfo) lumps. They are stored in the map separator (mapname) lump, which is normally empty. Each map can have its own MapInfo.

The WAD datafile format itself is somewhat constricting, mainly due to the 8 character lumpnames and lack of internal directory structure. This has been remedied in the WAD2/WAD3, PAK and PK3 datafile formats.

  • MapInfo lumps
  • Hexen/ZDoom-style MAPINFO support
  • WAD2/WAD3 support
  • ** PAK and PK3 support

Other new features

  • Autoexpanding Zone Memory system. Never run out of memory, until you REALLY run out of memory.
  • Second level dedicated cache system for sounds *etc.
  • * Can run several maps simultaneously
  • New game objects with custom AI are easy to make through inheritance
  • Level ordering is free, one level can in principle have any number of exits that lead to different levels.
  • Damage system with different types of damage
  • ** DLL modifiability

MapInfo lump structure

 The MapInfo lump is plain ASCII text and consists of blocks. Each block begins with 
[this is a blockname]<\n>
and ends when the next block begins, or when the lump ends.

The recognized blocknames are [map info] / [level info] [scripts] [intertext]
